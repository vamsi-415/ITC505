Addendum - Process and Creative Choices

Process: The initial step was to structure the HTML to create a simple and accessible interface. This included a start button, the game grid, a moves counter, and a help dialog. The CSS was created to ensure a clean and responsive design. A grid layout was used to arrange the tiles, and a button style was implemented for a cohesive look. JavaScript was used to handle the game logic, including starting the game, generating random patterns, handling tile clicks, and checking for a win condition. CSS classes (is-off) were used to manage tile states, allowing for easy toggling and styling without directly manipulating styles in JavaScript.

Creative Choices: A 5x5 grid was chosen for simplicity and clarity. The layout is easily scalable for larger grids. White and black were used for on and off states, providing a clear visual distinction. The Roboto font was selected for a modern and clean appearance. Ensuring the game works well on different devices and screen sizes was a priority.

Challenges: Ensuring that the correct surrounding tiles toggle correctly was a bit challenging. This was addressed by carefully checking the position of each tile (middle, edge, corner) and toggling only the valid surrounding tiles. Counting the "off" tiles to determine if the game is won needed a clear and efficient approach. Iterating through all tiles and checking their state solved this.

Insights Gained: Keeping CSS and JavaScript separate for styling and functionality, respectively, makes the code more maintainable. Using CSS Grid is a powerful way to handle game layouts. Adding event listeners efficiently and ensuring they work correctly across different elements enhances user interaction.
